# 0.4 — Relative CRPS (short-window scoring)

## Motivation

CRPS computed against ground-truth **absolute pose** is structurally
insensitive to σ calibration when the filter is overconfident, which is
the operating regime of every LIO algorithm we measured. Concretely, the
trans_crps score reported by `smfeval score` did not move under per-scan
σ inflation that brought local `mean(z²)` from 15→3 on christ-church-03
and 20→3 on blenheim-palace-05.

The reason is standard: for Gaussian `N(μ, σ²)` against truth `y`,

    CRPS = σ · [ z·(2Φ(z) − 1) + 2φ(z) − 1/√π ],   z = (y − μ)/σ

When `|z| ≫ 1` (overconfident) CRPS → `|y − μ|` — just the absolute
error. With σ ≈ 1.4 mm and APE ≈ 286 mm we get z ≈ 200; CRPS is in the
linear regime and any change in σ that keeps `|z| ≫ 1` is invisible.

This is wrong as a calibration metric for an *odometry* filter, whose
σ_post is a **local** precision quantity (bounded by the steady-state
Kalman gain) and is never expected to track integrated APE.

## What to add

A new metric: **relative CRPS** at window Δt, using the *relative pose
error* between consecutive (or Δt-spaced) frames:

    Δμ_est = pose_est(t + Δt) ⊖ pose_est(t)        # SE(3) increment
    Δμ_gt  = pose_gt (t + Δt) ⊖ pose_gt (t)
    Σ_rel  = Σ_post(t) + Σ_post(t+Δt)              # iid bound (or use
                                                    # propagated cross-cov)
    CRPS_rel = ∫ CRPS( N(Δμ_est, Σ_rel), Δμ_gt ) dt / N

Δt should default to one scan-period (0.1 s for 10 Hz LIO) and probably
also be reported at 1 s and 10 s to expose drift profile.

Why this is the right metric:

- σ_post tracks **local** uncertainty (bounded). |RPE| at short Δt is
  also local. z = |RPE|/σ_rel can be O(1) → CRPS in the *nonlinear*
  regime → sensitive to σ calibration.
- Long-Δt RPE-CRPS exposes drift / prior calibration as a separate
  axis from local precision.

## Implementation sketch

- Extend `smfeval/src/scoring/` with `relative.py` (mirror of the
  existing absolute scorer).
- CLI: `smfeval score --rpe-window 0.1 …` (or a list `--rpe-window
  0.1,1,10`).
- Reuse existing `--align`, `--sync`, `--body-frame-transform`; the
  relative formulation is alignment-invariant so for *pose* CRPS the
  align stage can be skipped, but for `gaussian_se3` we still want
  consistent body-frame convention.

## Reference implementation

`slam_benchmark/scripts/coverage_belief_rpe.py` already computes
`mean(z²)` at multiple Δt and uses the same `√(σ_i² + σ_j²)` bound. It
should be straightforward to lift the integral form here.

## Reference findings (slam_benchmark, 2026-05-27)

On `fast_lio2_belief` with per-scan bootstrap-`c` calibration:

| Δt    | cc03 mean(z²) naive | cc03 mean(z²) per-scan | bp05 mean(z²) naive | bp05 mean(z²) per-scan |
|-------|---------------------|------------------------|---------------------|------------------------|
| 0.1 s | 15.4                | **3.8**                | 19.9                | **3.4**                |
| 1 s   | 30.3                | 8.2                    | 94.6                | 17.5                   |
| 10 s  | 108.7               | 28.9                   | 1 835               | 359                    |

trans_z (current smfeval absolute) moved 2.2× with this calibration.
trans_crps did **not** move at all. Relative CRPS is what would have
caught the improvement.

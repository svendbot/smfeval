# SQUARE/0.4 — Distributional Ground Truth

Status: planned. Tag: `0.5` (tool version; the SQUARE format version this
plan introduces remains `SQUARE/0.4` — format and tool versions are
decoupled as of tool v0.4.0, which shipped with the format still at
`SQUARE/0.3`).

## Motivation

Treating GT as deterministic misrepresents SLAM evaluation. GT is itself a
measurement from an external observer (motion capture, RTK GPS, total
station, surveyed prior) with non-zero uncertainty. Every proper scoring
rule is defined as `E_{y∼P}[s(y, Q)]` where `P` is the GT distribution and
`Q` is the predictive — collapsing `P` to a delta is a special case, not
the general case.

Current code accesses `s.translation` / `s.quat_xyzw` on every GT step,
which works only for `DeterministicStep` / `GaussianStep`-as-point. An
ensemble GT crashes at runtime; a Gaussian GT silently ignores `Σ_P`. This
plan removes the assumption.

## Scope of the version bump

- **Format-side**: no header field changes. GT files have always been
  allowed to declare any representation; we just hadn't honored it. The
  bump signals semantic compatibility, not syntax change.
- **CLI-side**: `--gt` accepts any SQUARE representation. TUM GT continues
  to map to deterministic (`Σ_P = 0`).
- **Scoring-side**: every scorer takes `gt: Step` and dispatches on shape.

The 0.3 → 0.4 step is justified by behavior change (existing GT files
will score differently if they declare uncertainty), not file syntax.

## Closed forms (P, Q both Gaussian)

### Log score

`E_{y∼P}[−log q(y)]` for Gaussians:

```
= ½(μ_P − μ_Q)ᵀ Σ_Q⁻¹ (μ_P − μ_Q)    ← Mahalanobis at GT mean (existing)
+ ½ tr(Σ_Q⁻¹ Σ_P)                     ← NEW: GT variance term
+ ½ log det(2π Σ_Q)                   ← log-det normalizer (existing)
```

One-line addition to `gaussian_log_score`. `Σ_P = 0` (deterministic GT)
makes the trace term vanish, recovering current behavior exactly.

### CRPS (per-dimension)

For univariate Gaussians, generalize variance inflation:

```
CRPS_per_dim(Q, P) = σ_D[2φ(z) + z(2Φ(z) − 1)] − σ_Q/√π
where σ_D² = σ_Q² + σ_P²,  z = (μ_Q − μ_P) / σ_D
```

Existing Gaussian-vs-point formula generalizes by `σ_Q² → σ_Q² + σ_P²`.
Sum across translation dims; rotation uses angle-axis tangent.

### Interval score

For α-level `[l, u] = [μ_Q − z·σ_Q, μ_Q + z·σ_Q]`:

```
E_{y∼P}[IS_α] = (u − l)
              + (2/α)[(l − μ_P)·Φ((l − μ_P)/σ_P) + σ_P·φ((l − μ_P)/σ_P)]
              + (2/α)[(μ_P − u)·Φ((μ_P − u)/σ_P) + σ_P·φ((u − μ_P)/σ_P)]
```

Truncated-Gaussian moments. Closed form in terms of standard `Φ`, `φ`.

### No closed form (MC fallback)

- **Multivariate energy score**: `E‖X − Y‖` in `d > 1` has no closed form
  even for two Gaussians (Bessel functions or 1-D numerical integration).
- **Any ensemble Q or P**: empirical distribution, MC dominates.
- **PIT**: tractable in principle for Gaussian–Gaussian, but messy.

For all of these: Monte Carlo with shared sampling between terms.

## Variance reduction for MC paths

To bake in during PR3 (or alongside PR1/PR2 when touching the same files):

- **Common random numbers (CRN)**: when computing `E‖X − Y‖ − ½ E‖X − X'‖`,
  use the *same* X samples for both terms. Verify this holds after the
  GT-sampling addition (it currently does for the `−½ E‖X − X'‖` term).
- **Antithetic sampling**: for Gaussian Q (and P), draw standard normals
  `ε_i` and use both `μ + Lε_i` and `μ − Lε_i`. Halves variance on
  symmetric integrands at no extra cost.
- **Stratified quantiles**: for 1-D / per-dim MC (rotation CRPS,
  translation-magnitude interval score), use evenly-spaced quantile
  points of Q instead of random draws.
- **Quasi-Monte Carlo (Sobol)**: for 6-D tangent integrals (energy
  score), `scipy.stats.qmc.Sobol` reliably beats random samples for
  smooth integrands. Modest change.
- **Closed form as control variate**: where a closed form exists for a
  *related* quantity, use it to reduce variance of the MC estimator of
  the actual one. E.g. Gaussian-Q + Gaussian-P CRPS as CV for Gaussian-Q
  + ensemble-P CRPS.

These are pure efficiency wins. Defer until MC paths are otherwise
stable, then sweep them in as PR3.

## Open questions (resolve before coding)

1. **Calibration semantics with distributional GT.** PIT requires a
   point. Options:
   - Draw `y_i ∼ P` per timestep, compute `F_Q(y_i)`, aggregate. Nominal
     PIT distribution is no longer uniform — it's `F_Q ∘ P`. KS test
     reference changes.
   - Keep PIT at the GT mean (current behavior) and add a separate
     diagnostic for GT uncertainty.
   - Needs a research call. Defer.

2. **Interpolated GT** (`--sync=interpolate_gt`). The GP predictive
   variance is currently a *risk* signal, not GT noise. Should it now
   feed into `Σ_P`? Conceptually yes; practically it changes every
   score under interpolation. Decide: opt-in flag, or always-on?

3. **TUM GT promotion.** `TumHeader.to_square()` currently produces
   `representation=deterministic`, `gauge=fixed`, no covariance. That's
   correct for TUM (no uncertainty on disk); no change needed.

## Backward compatibility

- TUM GT: behavior identical (deterministic → `Σ_P = 0` → trace term
  vanishes, variance inflation no-op, etc.).
- SQUARE/0.3 deterministic GT: behavior identical.
- SQUARE/0.3 Gaussian or ensemble GT: previously crashed; now scored
  honestly. This is the breaking behavior change that justifies the
  minor bump.

Regression fixtures (`tests/fixtures/regression/`) with deterministic GT
should produce byte-identical reports across the 0.3 → 0.4 boundary. New
fixtures exercising Gaussian/ensemble GT can be added in PR2.
